# 5572 移动端独立前端架构设计

## [S1] 问题

当前5572网站使用响应式设计适配移动端，存在以下问题：
- 桌面端UI在移动端体验差（触摸目标小、布局拥挤、交互不友好）
- 响应式代码导致维护困难，容易出现各种适配问题
- 无法针对移动端特性优化（手势操作、底部导航、大触摸目标等）

## [S2] 解决方案

采用**共享核心+设备UI**架构：
- Core层：100%共享业务逻辑（API、Hooks、Store、Utils）
- UI层：设备特定组件（Desktop/Mobile/Tablet/TV）
- 路由层：根据设备类型加载不同UI

## [S3] 架构设计

### 目录结构
```
src/
├── core/                    # 共享核心
│   ├── api/                 # API调用（所有设备共享）
│   ├── hooks/               # 共享Hooks
│   ├── store/               # 状态管理
│   └── utils/               # 工具函数
│
├── ui/                      # 设备特定UI
│   ├── desktop/             # 桌面端组件
│   ├── mobile/              # 移动端组件
│   │   ├── components/      # 移动端专用组件
│   │   ├── layouts/         # 移动端布局
│   │   └── hooks/           # 移动端专用Hooks
│   ├── tablet/              # 平板端组件
│   └── tv/                  # TV端组件
│
├── app/                     # 路由页面
│   ├── layout.tsx           # 根布局（根据设备加载不同UI）
│   ├── page.tsx             # 首页
│   └── ...
```

## [S4] 移动端UI设计规范

### 布局原则
- 单列布局（不用网格）
- 底部Tab导航（不用顶部）
- 大触摸目标（44px+）
- 简化信息密度
- 底部弹窗（不用模态框）

### 交互原则
- 手势操作（滑动、长按）
- 触觉反馈
- 简化操作流程
- 减少输入需求

### 视觉原则
- 深色主题（#0a0a0a背景）
- 金黄主色（#f4c24d）
- 大图标、大文字
- 圆角卡片（16-24px）
- 微妙动画（300ms）
