# 5572影视 系统性整改方案

## 一、问题诊断

### 核心问题

1. **数据源混乱**：首页用豆瓣（信息源），播放用采集源，导致断裂
2. **平台不统一**：PC/手机/TV/APP 各自为政，功能不一致
3. **UI/UX碎片化**：页面跳转混乱，用户体验割裂
4. **资源未本地化**：海报、演员信息依赖豆瓣API

### 根本原因

- 架构设计时没有统一规划
- Douban被错误地用作视频源
- 各平台独立开发，缺乏统一标准

---

## 二、整改目标

### 短期（1-2周）

1. ✅ 修复播放功能 - 确保所有平台能正常播放
2. ✅ 统一数据源 - 首页从采集源获取内容
3. ✅ 本地化海报/演员 - 减少对豆瓣依赖

### 中期（2-4周）

1. 统一UI/UX体系 - PC/手机/TV一致体验
2. 完善PWA功能 - 离线缓存、推送通知
3. 优化播放器 - 兼容性、性能

### 长期（1-2月）

1. 完整的产品体系 - 所有平台功能一致
2. 高级功能 - AI推荐、社交互动
3. 性能优化 - 加载速度、响应时间

---

## 三、具体整改项

### 3.1 数据源统一（优先级：高）

**问题**：首页展示豆瓣数据，播放用采集源

**方案**：

```
首页数据源改造：
1. 从采集源获取热门/推荐内容
2. Douban仅用于：海报URL、演员信息、评分
3. 播放直接使用采集源，不经过Douban
```

**实施步骤**：

- [ ] 修改 `useHomePageQueries` 从采集源获取数据
- [ ] 添加采集源热门/分类API
- [ ] 首页展示采集源内容
- [ ] Douban降级为补充信息源

---

### 3.2 资源本地化（优先级：高）

**问题**：海报、演员信息依赖豆瓣API

**方案**：

```
本地化策略：
1. 海报：首次访问时缓存到本地
2. 演员信息：存储到数据库
3. 评分/评论：定期同步
```

**实施步骤**：

- [ ] 海报缓存机制（已有部分实现）
- [ ] 演员信息数据库设计
- [ ] 定期同步脚本
- [ ] 离线访问支持

---

### 3.3 平台统一（优先级：高）

**问题**：PC/手机/TV/APP 各自为政

**方案**：

```
统一架构：
1. 核心逻辑共享（数据、API、播放器配置）
2. UI层独立（响应式设计）
3. 功能对齐（各平台功能一致）
```

**实施步骤**：

- [ ] 抽取核心业务逻辑到共享层
- [ ] 统一API接口
- [ ] 播放器配置标准化
- [ ] 功能清单对齐

---

### 3.4 UI/UX统一（优先级：中）

**问题**：页面跳转混乱，体验割裂

**方案**：

```
统一设计系统：
1. 组件库标准化
2. 导航逻辑统一
3. 响应式设计规范
4. 交互模式一致
```

**实施步骤**：

- [ ] 设计系统文档
- [ ] 组件库重构
- [ ] 导航逻辑统一
- [ ] 响应式断点优化

---

### 3.5 播放器兼容（优先级：中）

**问题**：各平台播放器不一致

**方案**：

```
播放器统一：
1. Web: ArtPlayer + HLS.js（已实现）
2. APP: media_kit（已实现）
3. 配置标准化
4. 兼容性测试
```

**实施步骤**：

- [ ] 播放器配置文档
- [ ] 跨平台测试
- [ ] 性能优化
- [ ] 错误处理统一
